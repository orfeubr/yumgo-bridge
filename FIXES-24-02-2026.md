## ✅ 4. yumgo.com.br - 404 Corrigido

**Problema**: https://yumgo.com.br retornava 404
**Causa**: Dois problemas:
1. Nginx não estava configurado para aceitar yumgo.com.br
2. Rota duplicada em tenant.php causava conflito

**Soluções aplicadas**:

### a) Nginx - Adicionado domínio

```nginx
# ANTES
server_name food.eliseus.com.br *.eliseus.com.br;

# DEPOIS
server_name food.eliseus.com.br *.eliseus.com.br yumgo.com.br *.yumgo.com.br;
```

**Comandos executados**:
```bash
sudo sed -i 's/server_name food.eliseus.com.br \*.eliseus.com.br;/server_name food.eliseus.com.br *.eliseus.com.br yumgo.com.br *.yumgo.com.br;/' /etc/nginx/sites-enabled/food
sudo nginx -t
sudo systemctl reload nginx
```

### b) Rota Duplicada - Removida de tenant.php

O problema era que havia DUAS rotas para `/`:
- `routes/web.php` - trata domínios centrais E tenants
- `routes/tenant.php` - tinha outra rota `/` que sobrescrevia

A rota de tenant.php tinha middleware `PreventAccessFromCentralDomains` que bloqueava yumgo.com.br com 404.

**Arquivos modificados**:
- `/var/www/restaurante/routes/tenant.php` (linha 53 - comentada)

```php
// ANTES
Route::get('/', [\App\Http\Controllers\RestaurantHomeController::class, 'index'])->name('catalog');

// DEPOIS
// REMOVIDO: já tratado em web.php para permitir domínios centrais mostrarem lista de restaurantes
// Route::get('/', [\App\Http\Controllers\RestaurantHomeController::class, 'index'])->name('catalog');
```

**Resultado**: Agora yumgo.com.br mostra a lista de restaurantes parceiros! ✨
